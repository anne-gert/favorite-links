Favorites and ToDo - Multiple Configurations

Just plain favorites.html without URL arguments, creates a single, default configuration that is stored in the browser's local storage. Using a configuration name with ?links=<configuration-name> also allows the possibility to store multiple configurations in the browser's local storage, each under its own name. See URL arguments for all the arguments.

As an example, go to http://example.net/favorites.html?items=this-is-a and enter the following configuration under Settings (⚙): 

!title This is the page for config A

http://example.net/favorites.html?links=this-is-a
http://example.net/favorites.html?links=this-is-b
Click the second link and enter the following configuration: 
!title This is the page for config B

http://example.net/favorites.html?links=this-is-a
http://example.net/favorites.html?links=this-is-b

Now, both configurations are independently available.
