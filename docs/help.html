Favorites and ToDo - Help

Description

favorites.html is a single page that can be configured to show links and todo lists, organized in sections, blocks and items. This configuration can be downloaded via a URL and/or edited in the browser's local storage (see below).
This page can be set as home page in the browser's settings and as the Tab start page via an add-on like New Tab Override.

Note that the home page can be a local file (with file:///... URL) or an internet address (with http://... or https://... URL) and that the tab start page can only be an internet address.

Standard Usage

The standard usage is to open the page favorites.html (or possibly set it as your home page). Upon first opening, it shows a default example page. This page can be edited by any of the following ways:

  • Click Settings (gears icon, ⚙) at the top right of the page. This opens the Settings where the textual configuration can be edited and saved.
  • Double click the page near the spot where you want to add or change an item. This opens the Settings with the cursor on that line.
  • Drag-and-Drop URLs or text onto the page. This causes the items to be inserted on that spot (see Drag-and-Drop for more details).
Optionally, this configuration can be maintained in a separate text file and copy-pasted into the Settings.
Since GitHub provides for a way to serve HTML pages directly, this can be used to open and use favorites.html directly from GitHub. Save the target of this link as your home page and you're set.

Tip: By using favorites.html?links=<config-name>, multiple independent configurations can be defined, each having their own URL. Each configuration has its own load-options and save-options under Settings, but if those are left blank, the configuration is only stored locally (see Read/write online configuration for details).

Getting Started...

The easiest method to get started is to copy the contents of the example-links.txt file into the Settings text panel and start editing that.
The vast majority of the lines will simply consist of indentation, name and URL. Add a few !search options and that's it.
